# MCP 服务器与扩展能力设置

点击菜单进入“设置” -> “Mcp”，这里管理 Model Context Protocol (MCP) 服务器配置，允许外挂本地或远程的扩展能力（如查数据库、搜网页、浏览器控制等）。

## 1. 远程 HTTP 类型
适用于通过网络提供服务的扩展配置：
*   **Name (名称)**：别名标识。
*   **URL (接口链接)**：远程服务端的 Streamable HTTP 链接。
*   **Headers (请求头)**：可选填入认证 Token 等静态头部键值对。

## 2. 本地 stdio 类型
适用于在容器内通过本地命令行进程启动的服务：
*   **Name (名称)**：别名标识。
*   **Command (执行命令)**：执行引擎或命令，例如 `node`、`npx`、`python` 等。
*   **Args (命令参数)**：参数列表，例如 `/path/server.js` 或具体启动参数。
*   **Env (环境变量)**：传递给该进程的键值对环境变量。

*提示：对于复杂的 MCP 配置，AI 也可以在会话中通过 `manageMcp` 工具为用户自动添加或管理。*
