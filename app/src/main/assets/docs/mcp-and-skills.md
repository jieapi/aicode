# MCP 服务器与扩展能力设置

进入“设置” -> “MCP 服务器”，管理 Model Context Protocol (MCP) 服务器配置，外挂本地或远程扩展能力（如查数据库、搜网页、浏览器控制等）。

## 1. 服务器列表页
*   列出所有已配置的 server 卡片，左侧图标区分 stdio（终端图标）/ HTTP（服务器图标），右下角状态圆点（已连接/连接中/连接失败/已禁用）。
*   每张卡片显示：名称、状态 Pill、类型 Pill（`内置` 或 `HTTP`）、信息 Pill（已连接时显示「工具: N」，否则显示启动命令或 URL）。
*   **左滑卡片**露出「删除」按钮可移除该 server；点击卡片进入编辑。
*   右上角「重新连接」按钮一键重连所有 server，「+」按钮添加新 server。

## 2. 编辑器：基础设置 Tab
顶部「是否启用」开关控制该 server 是否参与连接。其余字段按传输类型切换（`远程 HTTP` / `本地 stdio`）：

### 远程 HTTP 类型
*   **名称**：别名标识（如 `fetch`、`github`）。
*   **服务器地址**：远程服务端的 Streamable HTTP 链接（如 `https://api.example.com/sse`）。
*   **自定义请求头**：键值对列表，可添加多条，用于填入认证 Token 等静态头部。

### 本地 stdio 类型
*   **名称**：别名标识。
*   **启动命令**：执行引擎或命令，例如 `npx`、`node`、`python`。
*   **命令参数**：参数列表，每项一个卡片（如 `-y`、`@modelcontextprotocol/server-xxx`）。
*   **环境变量**：键值对列表，传递给该进程的环境变量（如 `API_KEY`）。

## 3. 编辑器：工具 Tab
连接成功后可查看该 server 暴露的工具列表，每个工具一张卡片：
*   **工具开关**：右上角 Switch 可禁用单个工具（写入 `disabledTools`），禁用后该工具不再注入模型。
*   **描述与参数**：显示工具描述及其入参键名 pill。
*   **需要审批**：底部 Switch 开启后，该工具每次调用前需用户确认（写入 `requireApprovalTools`）。
*   未连接或未检测到工具时，点击右上角「刷新」按钮重新拉取工具列表。

## 4. 其他入口
*   编辑器右上角日志按钮可跳转到「日志查看」并按该 server 名称过滤。
*   对于复杂的 MCP 配置，AI 也可在会话中通过 `manageMcp` 工具自动添加或管理。
