# 远程工作区 (Remote Servers) 与同步设置

让手机通过 SFTP / FTP / 本地镜像连接远程或本地目录作为项目工作区。进入“设置” -> “远程工作区”，页面分 4 个 Tab。

## 1. 连接配置 (Tab 1)
点击「+」填写连接通道参数：
*   **协议类型**：FilterChip 三选一：`SFTP`（默认端口 22）/ `FTP`（默认端口 21）/ `本地`（LOCAL，端口置 0）。
*   **连接名称** / **通道名称**：别名（本地通道提示如“内部存储镜像”）。
*   **主机地址**：IP 或域名；本地通道则填“内部存储目标目录”（如 `/storage/emulated/0/AICode/projects`）。
*   **端口 / 用户名 / 密码**：SFTP/FTP 必填；密码框可切换可见性。本地通道不显示这些字段，工作区会单向同步到该目录，供只能访问普通目录的 App 使用。
*   **测试连通性 / 测试目录**：保存前可点击测试（本地通道测试目录可达）。

## 2. 工作区 (Tab 2)
点击「+」选择一个已建好的连接通道，建立映射关系：
*   **关联远程连接通道**：下拉选择连接。
*   **远程目标目录 / 镜像目标子目录**：右侧文件夹图标可打开**远程目录浏览器**逐级浏览选择；本地通道填根目录下子目录，通常保持 `/`。
*   **映射到本地工作区**：下拉选择本地工作区（需先在侧边栏创建）。
*   **应用启动时自动连接**：开关，开启后开机自动连接并同步该工作区。
*   连接后卡片提供「断开连接」「上传全部 / 同步全部」「下载全部」（本地通道无下载）与「连接并同步」。

## 3. 内置FTP (Tab 3) - 内置 FTP 服务端
手机端内置轻量级 FTP 服务器，方便电脑端远程管理手机工作区：
*   **使用说明**：手机与电脑需连同一 WiFi；开启后在电脑文件资源管理器或 FileZilla 地址栏访问显示的 FTP 地址（如 `ftp://192.168.1.100:2121`）。
*   **服务开关**：开启后显示“运行中: $serverUrl”，关闭显示“服务未开启”。
*   **服务参数配置**：监听端口、登录用户名、登录密码（开匿时禁用）、「允许匿名访问」开关、「开机自启」开关，底部「保存配置」按钮。
*   连接后可访问并管理手机 `projects/` 目录下的全部代码项目。

## 4. 同步设置 (Tab 4)
配置底层文件同步引擎的行为：
*   **目录/文件忽略列表**：多个规则用英文逗号分隔，匹配的目录及文件不会被同步（如 `node_modules`），底部「保存规则」。
*   **遵循 .gitignore 规则**：开关，开启后自动读取项目根目录 `.gitignore` 进行过滤。
*   **单次最大同步队列数量**：防止突发大量文件修改同时请求服务端断连，超过此数量的操作排队按批次执行（批次间稍作停顿），底部「保存数量」。
