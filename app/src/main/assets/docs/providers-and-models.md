# AI 提供商 (Providers) 与模型配置指南

点击菜单进入“设置” -> “Providers”，可以添加或修改大模型 API 配置，包含以下具体字段：

## 1. 配置字段说明
*   **名称 (Name)**：文本框。给提供商取个别名（如“我的中转 API”）。
*   **类型 (Type)**：选项框。一般为 `openai` 或 `anthropic`。决定发包协议。第三方中转站基本都选 `openai`。
*   **API Key**：文本框。填入秘钥（如 `sk-xxx`）。
*   **Base URL (接口地址)**：文本框。填 API 根域名（如 `https://api.openai.com`）。不要带尾部的 `/v1` 或具体的 path。
*   **API Path (接口路径)**：文本框。紧跟 Base URL 后的请求路径，默认是 `/chat/completions`。
*   **Use Response API**：开关。非标准解析模式，默认**关闭**。

## 2. 模型管理
*   **模型 (Models)**：可以点击拉取按钮去同步提供商提供的模型列表，也可以手动输入（如 `gpt-4o`, `claude-3-5-sonnet`）后点击添加。
*   **默认模型 (Default Model)**：下拉框。选择该提供商默认选用的出战模型。

## 3. 识图专用模型
进入「设置」->「识图模型」，可为识图（`viewImage` 工具）单独指定一个**兜底模型**：

*   **跟随当前聊天模型**（默认）：识图始终用当前选中的聊天模型。当该模型本身支持图片输入时，直接用其原生视觉能力识别图片；若不支持图片输入，则识图不可用并报错。
*   **指定专用模型**：选择某个提供商下、支持图片输入的模型。当当前聊天模型不支持图片输入时，识图那一轮会**临时切换**到该专用模型发送请求，发送完成后自动恢复回聊天模型继续对话。

要点：
*   识图**优先**用当前聊天模型的原生能力；专用模型仅作"当前模型不支持图片"时的兜底，不会在当前模型已支持图片时被使用。
*   列表中每个模型旁标注是否「支持图片输入」（依据模型元数据 `supportsVision`），便于挑选；即便选了不支持图片的模型也会保存，但实际识图发送时会被工作流校验并报错。
*   若当前聊天模型不支持图片、且专用模型未配置或同样不支持图片，`viewImage` 将返回 `MODEL_VISION_UNSUPPORTED`，提示去设置中指定识图模型。
