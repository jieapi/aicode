# AI 提供商 (Providers) 与模型配置指南

进入“设置” -> “AI 提供商”，可添加或修改大模型 API 配置。提供商编辑页分两个 Tab：**配置** 与 **模型**。

## 1. 配置 Tab 字段
*   **名称**：文本框。给提供商取个别名（如“我的中转 API”）。留空保存时自动填“新提供商”。
*   **类型**：FilterChip 三选一：`OpenAI` / `Anthropic` / `Gemini`。决定发包协议。第三方中转站基本都选 `OpenAI`。
*   **API Key**：文本框。填入秘钥（如 `sk-xxx`）。
*   **Base URL**：文本框。填 API 根域名，placeholder 随类型变化（OpenAI→`https://api.openai.com/`，Anthropic→`https://api.anthropic.com/`，Gemini→`https://generativelanguage.googleapis.com/`）。留空时按类型回填默认值。不要带尾部的 `/v1` 或具体的 path。
*   **API 地址**：文本框。紧跟 Base URL 后的请求路径，默认 `/chat/completions`。
*   **Response API (新版)**：开关，仅当类型为 `OpenAI` 时显示。默认**关闭**。

## 2. 模型 Tab
*   **拉取模型**：点击“拉取模型”按钮，向提供商同步可用模型列表；结果按品牌分组、可折叠，支持名称搜索筛选，每个模型旁标注元数据标签（`Chat` / `Image`(支持图片) / `Tools`(支持工具调用) / `Input <上下文>` / `Output <最大输出>`）。
*   **手动添加模型**：点击右上角 + 或“添加模型”，输入模型名（如 `gpt-4o`）即可加入。重复添加会被拦截提示。
*   **测试**：每个已添加模型行有“测试”按钮，对该模型发起一次连通性请求，行下方显示成功/失败结果。
*   **删除**：模型行右侧 × 可移除该模型。
*   **生效模型**：实际出战模型取 `selectedModel`，为空时回退到 `defaultModel`（即 `effectiveModel`）。提供商列表页可切换启用/停用某个提供商。

## 3. 识图专用模型
进入「设置」->「识图模型」，可为识图（`viewImage` 工具）单独指定一个**兜底模型**。列表按提供商分组，仅列出已启用且已添加模型的提供商：

*   **跟随当前聊天模型**（默认，列表首项）：识图始终用当前选中的聊天模型。当该模型本身支持图片输入时，直接用其原生视觉能力识别图片；若不支持图片输入，则识图不可用并报错。
*   **指定专用模型**：选择某个提供商下、支持图片输入的模型。当当前聊天模型不支持图片输入时，识图那一轮会**临时切换**到该专用模型发送请求，发送完成后自动恢复回聊天模型继续对话。

要点：
*   识图**优先**用当前聊天模型的原生能力；专用模型仅作“当前模型不支持图片”时的兜底，不会在当前模型已支持图片时被使用。
*   列表中每个模型旁标注是否「支持图片输入」/「不支持图片输入（不推荐用于识图）」（依据模型元数据 `supportsVision`）；即便选了不支持图片的模型也会保存，但实际识图发送时会被工作流校验并报错。
*   若当前聊天模型不支持图片、且专用模型未配置或同样不支持图片，`viewImage` 将返回 `MODEL_VISION_UNSUPPORTED`，提示去设置中指定识图模型。
