## 1. 顶部标题栏 (ChatHeader)
App 最顶部是聊天页标题栏，包含：
*   **菜单按钮**（左上）：打开侧边抽屉，管理会话列表、新建会话。
*   **会话标题**：当前会话名称。
*   **模型名**：当前启用的模型名称。
*   **终端入口**：跳转到终端页。
*   **Git 入口**：跳转到 Git 版本管理页。
*   **新建会话**：开始一个新的聊天。

---

## 2. 聊天输入框下方工具栏
输入框下方一行是操作工具栏，从左到右依次为：
*   **模式按钮**：点击在 BUILD → PLAN → AUTO 间循环切换（详见下节）。
*   **模型按钮**：显示当前启用的 AI 模型，点击可快速切换模型或跳转到设置管理提供商。
*   **工作区按钮**：显示当前工作区名称，点击弹出工作区管理面板（底部弹窗）：
    *   列出所有已创建的工作区，点击切换。若当前有正在运行的 AI 会话或终端标签，会弹出确认框提示切换会停止它们。
    *   可新建工作区（输入名称创建）。
    *   可删除已有工作区。
    *   工作区是 AI 操作的代码项目根目录，每个工作区对应手机上的一个独立文件夹，AI 的代码读写和命令执行都被沙盒隔离在当前工作区内。
*   **文件按钮**（文件图标）：上传文件作为附件，AI 可读取其内容。非忙碌时可用。
*   **图片按钮**（图片图标）：上传图片作为附件，AI 可识别图片内容。需当前模型支持图片输入，非忙碌时可用。
*   **发送按钮**：发送当前输入；生成中变为停止按钮。
*   上传的附件会以预览卡形式显示在输入框上方，可移除后再发送。

## 3. 核心模式切换 (Plan / Build / Auto)
工具栏左侧的模式按钮点击在三种模式间循环切换：**BUILD → PLAN → AUTO → BUILD**。
*   **BUILD (构建模式)**：全功能模式，解锁写权限。AI 可以帮用户写文件、修改代码以及运行构建命令。
*   **PLAN (计划模式)**：进入此模式后，系统物理级锁死 AI 的写操作权限。AI 只能进行代码阅读、检索和方案构思，绝不会修改任何文件。
*   **AUTO (自动模式)**：权限全放行模式。所有工具调用（写文件、执行命令、终端操作等）自动批准，不弹权限确认窗。仅灾难性删除（`rm` 根目录/系统目录等）仍被拦截。该模式权限极大，适合用户已明确知晓风险、希望 AI 全自动完成批量任务的场景。
*   *提示：除了用户手动点按钮切换，AI 自己在规划清晰后，也可以通过调用 `switchMode` 工具向用户发送切换模式的请求，用户在界面上一键批准即可。注意：AUTO 模式只能由用户在界面上手动进入，AI 无法通过工具切换到 AUTO；但处于 AUTO 模式时，AI 可通过 `switchMode` 切换到 PLAN 模式退出自动模式（这是 AI 退出 AUTO 的唯一路径）。*

## 4. 设置首页菜单项
“设置”首页以卡片列表呈现各二级入口，自上而下依次为：
*   **AI 提供商**：显示已添加数量与当前启用提供商；进入「AI 提供商」二级页管理（详见 providers-and-models.md）。
*   **识图模型**：显示“跟随当前聊天模型”或专用提供商·模型；进入「识图模型」二级页。
*   **MCP 服务器**：显示已配置数量与已连接数；进入「MCP 服务器」二级页（详见 mcp-and-skills.md）。
*   **容器镜像**：显示当前镜像 profile 名（默认“内置 Alpine”）；进入「容器镜像」二级页（详见 container-image.md）。
*   **日志等级**：显示当前等级名；进入「日志等级」二级页选择等级。
*   **日志查看**：查看最近日志，支持 MCP 名称过滤；进入「日志查看」二级页。
*   **工具授权**：显示已保存授权规则条数；进入「工具授权」二级页。
*   **远程工作区**：管理 SFTP / FTP 工作区同步；进入「远程工作区」二级页（详见 remote-servers.md）。
*   **外观主题**：首页内联下拉框，自动/深色/浅色。
*   **后台运行保活**：首页内联开关，开启后显示前台通知避免后台被杀。
*   **备份与还原**：加密导出/导入配置、聊天历史与凭据；进入「备份与还原」二级页（详见 backup-and-restore.md）。
*   **关于**：版本检查 · GitHub · 许可证；进入「关于」二级页。

## 5. 外观主题
“设置”首页「外观主题」行内联下拉框切换 App 明暗外观：
*   **自动**：跟随系统当前的明暗主题。
*   **深色**：固定使用深色主题。
*   **浅色**：固定使用浅色主题。
*   该设置会被保存，重启 App 后仍然生效。

---

## 6. 关于 (About)
“设置”首页「关于」入口（图标为信息圆圈）。这是一个纯展示型页面，无任何持久化设置，点击进入后展示下列信息：
*   **应用信息**：顶部卡片左侧显示 App 图标，右上为应用名 `AiCode`，右下为一句简介。
*   **版本（点击检查更新）**：单独一行卡片，显示当前版本号 `v<versionName>`（通过系统 PackageManager 读取，即对外发布版本，与 git tag `v<versionName>` 一致）。点击该行会自动通过 GitHub API `https://api.github.com/repos/jieapi/aicode/releases/latest` 查询最新 Release 的 tag，与当前版本号比对：
    *   相等 → 弹窗提示「已经是最新版本」。
    *   不同 → 弹窗「发现新版本 vX」，点击「前往下载」用浏览器打开 GitHub Releases 页面获取最新 APK。
    *   网络失败或解析失败 → 弹窗提示「检查失败」并给出错误信息。
*   **GitHub 仓库**：点击用系统浏览器打开本项目源码仓库 `https://github.com/jieapi/aicode`。
*   **开源许可证**：本项目使用 **GPL-3.0**，点击用浏览器打开完整 LICENSE 文件。
